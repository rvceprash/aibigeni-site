How it works

User query → Dialogflow CX (intent & entities)
→ Orchestration API (FastAPI on Cloud Run)
→ Vertex AI Search over flight inventory / cached offers
→ Pricing & rules engine → Options ranked by price, duration, stops
→ AI explains tradeoffs → user approves → Stripe checkout
→ Confirmation + itinerary.
  • Built components today: Dialogflow CX intents, Vertex AI Search over hotels (flights index in progress), Stripe Elements checkout, Cloud Run backend, React+Vite frontend.
  • Near‑term: Firestore persistence, Discovery Engine configured to BigQuery flight/hotel tables, CX parameter hardening, live inventory integration.
